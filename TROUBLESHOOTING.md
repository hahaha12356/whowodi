# 微信小程序运行配置说明

## ⚠️ 重要提示

### CSS兼容性问题
本项目使用了以下CSS属性，需要微信基础库版本支持：
- `gap` 属性：需要基础库 **2.7.1+**
- SCSS Mixins：需要正确导入

### 已修复的问题
✅ **修复**: `pages/history/history.vue` 中 `@include flex-center` 未定义
- 已添加 `@import '@/styles/mixins.scss';`

## 🚀 运行步骤

### 1. 安装依赖
```bash
npm install
```

### 2. 开发模式
```bash
npm run dev:mp-weixin
```

### 3. 微信开发者工具设置

#### 基础库版本
确保微信开发者工具的基础库版本 >= **2.7.1**

**设置路径**：
- 微信开发者工具 → 详情 → 本地设置
- 调试基础库：选择 >= 2.7.1 的版本

#### 项目配置
- ES6转ES5：✅ 勾选
- 增强编译：✅ 勾选
- 使用npm模块：✅ 勾选

### 4. 如果仍出现错误

#### 错误1: "Undefined mixin"
**原因**：页面中使用了mixin但未导入

**解决方案**：在对应页面的 `<style>` 标签中添加：
```scss
<style lang="scss" scoped>
@import '@/styles/mixins.scss';
// ... 其他样式
</style>
```

#### 错误2: "gap属性不生效"
**原因**：微信基础库版本过低

**解决方案**：
1. 升级微信开发者工具
2. 在项目设置中提高调试基础库版本到 2.7.1 或更高

## 🔧 调试建议

### 1. 清除缓存
如果遇到奇怪的错误，尝试：
- 微信开发者工具 → 清除缓存
- 删除 `node_modules` 和 `dist` 目录
- 重新 `npm install`
- 重新 `npm run dev:mp-weixin`

### 2. 检查文件路径
确保所有import路径正确：
- `@/` 别名指向项目根目录
- 检查文件名大小写（Linux区分大小写）

### 3. 查看详细错误
在微信开发者工具的**控制台**查看详细错误信息

## 🎯 常见问题排查

### Q: 页面空白怎么办？
A:
1. 查看控制台错误信息
2. 检查所有import是否正确
3. 确认npm依赖已正确安装

### Q: 样式不生效？
A:
1. 检查SCSS是否正确编译
2. 清除缓存重新编译
3. 确认基础库版本 >= 2.7.1

### Q: 组件找不到？
A:
1. 检查组件路径是否正确
2. 确认组件文件存在
3. 检查组件是否正确导入

### Q: 点击按钮没反应？
A:
1. 查看控制台是否有JS错误
2. 检查事件绑定是否正确
3. 确认方法名拼写正确
