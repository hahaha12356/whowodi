# 并发领取身份修复 - 测试指南

## 快速测试步骤

### 准备工作

1. **编译项目**
   ```bash
   # 在微信开发者工具中点击"编译"按钮
   # 或者使用命令行
   npm run dev:mp-weixin
   ```

2. **检查云数据库权限**
   - 打开微信开发者工具
   - 点击"云开发" → "数据库"
   - 找到 `claims` 集合
   - 确保权限设置为：**所有用户可读、所有用户可写**

3. **准备测试设备**
   - 至少需要2台手机或2个微信账号
   - 建议使用3-5台设备进行更全面的测试

### 测试场景1：基础并发测试（2人）

**目标**：验证2个用户同时领取时不会出现重复

**步骤**：
1. 在设备A上创建一个游戏（2人，2个平民）
2. 点击"分享"，将链接发送给设备B
3. 设备A和设备B同时点击"领取身份"（尽量在1秒内）
4. 查看两个设备的身份号码

**预期结果**：
- ✅ 设备A：1号或2号
- ✅ 设备B：1号或2号（与设备A不同）
- ❌ 不应该出现两个设备都是1号或都是2号的情况

**日志检查**：
打开微信开发者工具的控制台，查看日志：
```
第1次尝试 - 随机选中座位: 1 (从 2 个空位中选择)
创建领取记录: XXXX 1 claimId: xxxxx
验证第1次: 座位1有1条记录
查询结果已稳定
✅ 第1次尝试 - 领取成功! 座位: 1
```

### 测试场景2：高并发测试（3-5人）

**目标**：验证多人同时领取时的稳定性

**步骤**：
1. 在设备A上创建一个游戏（5人，5个平民）
2. 将链接分享给其他4台设备
3. 5台设备同时点击"领取身份"（尽量在1秒内）
4. 查看所有设备的身份号码

**预期结果**：
- ✅ 5个设备分别获得1、2、3、4、5号（不重复）
- ✅ 可能会看到一些设备有重试的日志（这是正常的）

**日志检查**：
可能会看到冲突和重试的日志：
```
第1次尝试 - 随机选中座位: 2 (从 5 个空位中选择)
检测到并发冲突，座位 2 有 2 条记录
❌ 我是输家，删除记录并重试: XXXX 2 赢家ID: xxxxx
第2次尝试 - 随机选中座位: 4 (从 4 个空位中选择)
✅ 第2次尝试 - 领取成功! 座位: 4
```

### 测试场景3：重复领取测试

**目标**：验证用户不会重复领取

**步骤**：
1. 用户A领取身份（假设获得1号）
2. 用户A点击"隐藏"
3. 用户A再次点击"查看我的身份"

**预期结果**：
- ✅ 显示相同的身份号码（1号）
- ✅ 不会创建新的领取记录

**日志检查**：
```
从本地缓存恢复身份: 1
```
或
```
从云端恢复身份: 1
```

### 测试场景4：座位已满测试

**目标**：验证所有座位被占用后的提示

**步骤**：
1. 创建一个3人游戏
2. 3个用户都已领取身份
3. 第4个用户尝试领取

**预期结果**：
- ✅ 提示"所有座位已被占用"
- ✅ 不会创建新的领取记录

**日志检查**：
```
剩余空位数: 0 已占用: 3
所有座位已被占用
```

### 测试场景5：网络延迟模拟

**目标**：验证在网络较慢时的稳定性

**步骤**：
1. 在微信开发者工具中，打开"Network" → 选择"Slow 3G"
2. 创建游戏并尝试领取身份

**预期结果**：
- ✅ 可能需要更长时间，但最终能成功领取
- ✅ 可能会看到多次重试的日志

## 常见问题排查

### 问题1：仍然出现重复号码

**可能原因**：
1. 云数据库权限设置不正确
2. 代码未重新编译
3. 网络延迟过大

**解决方法**：
1. 检查 `claims` 集合权限
2. 重新编译项目
3. 增加验证时间（修改 `utils/cloud.js` 第263行的等待时间）

### 问题2：领取失败，提示"网络繁忙"

**可能原因**：
1. 云数据库未初始化
2. 网络连接问题
3. 超过最大重试次数

**解决方法**：
1. 检查 `App.vue` 中的云开发初始化代码
2. 检查网络连接
3. 增加最大重试次数（修改 `pages/view/view.vue` 第250行）

### 问题3：领取速度较慢

**可能原因**：
1. 验证时间较长（200ms + 4次验证 × 60ms = 约440ms）
2. 网络延迟

**解决方法**：
- 这是正常的，为了确保并发安全性，需要足够的验证时间
- 可以适当减少验证次数，但可能会增加冲突风险

## 性能监控

### 关键指标

1. **冲突率**：有多少次领取遇到了冲突
   - 查看日志中 "检测到并发冲突" 的次数
   - 正常情况下应该 < 20%

2. **重试次数**：平均每次领取需要重试多少次
   - 查看日志中 "第X次尝试" 的数字
   - 正常情况下应该 < 3次

3. **成功率**：最终成功领取的比例
   - 应该接近 100%

### 日志分析

在微信开发者工具的控制台中，可以看到详细的日志：

```
✅ 成功的日志：
第1次尝试 - 领取成功! 座位: 1

⚠️ 冲突的日志：
检测到并发冲突，座位 2 有 2 条记录
❌ 我是输家，删除记录并重试

❌ 失败的日志：
超过最大重试次数: 15
```

## 总结

通过以上测试，应该能够验证并发领取身份的修复是否有效。如果测试中发现问题，请查看日志并根据上述排查方法进行调试。
